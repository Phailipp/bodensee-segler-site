# Bodensee Segler (Side project)

This folder contains the HTML prototype you sent via WhatsApp.

## What it is
A premium single-page sailing companion with:
- Leaflet map + curated POIs (harbors, anchorages, rentals, gastro, services)
- **Multi-lake support** via `data/lakes.json` + `data/lakes/<slug>/*`
- A lake selector in the UI (shareable via `?lake=<slug>`)
- Lightweight **detail pages** (`place.html?...`) used from the modal

## Data verification pipeline (candidates â†’ verified)
All items in `data/*.json` now support:
- `candidateUrl`, `candidateFoundAt`, `candidateSource` (unverified)
- `url`, `source`, `lastVerified` (verified)

A Playwright-based verifier can promote `candidateUrl` to `url/source/lastVerified` **only** when the page is reachable and content looks plausible.

### Install (once)
```bash
cd side_projects/bodensee_segler
npm i
npx playwright install --with-deps chromium
```

### Verify N candidates (batch)
```bash
# verify up to 25 candidates (default)
npm run verify:candidates

# verify up to 10 candidates
npm run verify:candidates -- --limit 10

# dry-run (no writes)
npm run verify:candidates -- --dry-run 1
```

Or via wrapper:
```bash
./tools/verify_batch.sh 20
```

## Sitemap
